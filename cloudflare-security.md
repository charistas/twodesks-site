# Cloudflare Security Hardening

Last checked: 2026-07-05

These two sites are served from GitHub Pages behind Cloudflare, not from Cloudflare Pages static assets. A Cloudflare Pages `_headers` file will not affect the live responses while GitHub Pages remains the origin.

## Applied Cloudflare Settings

Applied on 2026-07-05 to both `tendijournal.app` and `twodesks.app`.

- Always Use HTTPS is enabled. Plain HTTP requests redirect to the same HTTPS URL.
- Minimum TLS version is set to TLS 1.2.
- TLS 1.3 remains enabled.
- HSTS is enabled with `max-age=2592000` (1 month), `includeSubDomains` off, and preload off.
- Cloudflare's HSTS No-Sniff option is enabled, which sends `X-Content-Type-Options: nosniff`.
- A host-scoped Response Header Transform Rule named `Security response headers` is active on each zone.
  - `tendijournal.app`: `(http.host in {"tendijournal.app" "www.tendijournal.app"})`
  - `twodesks.app`: `(http.host in {"twodesks.app" "www.twodesks.app"})`
  - Set `Referrer-Policy: strict-origin-when-cross-origin`.
  - Set `Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=(), usb=()`.
  - Set `X-Frame-Options: DENY`.
  - Remove `Access-Control-Allow-Origin`.

## Remaining Findings

- Cloudflare is still modifying live HTML:
  - Tendi email links are rewritten by Email Address Obfuscation.
  - Two Desks receives Cloudflare-injected bot/AI-protection markup and JavaScript.

## Deferred: Strict CSP

Do not add a strict CSP while Cloudflare is injecting or rewriting HTML.

Strict CSP path:

1. First disable Cloudflare features that inject or rewrite page HTML, especially Email Address Obfuscation, AI Labyrinth, and JavaScript-based bot detections. The site-owned email links are now generated by local JavaScript, so Cloudflare Email Address Obfuscation is not needed for these pages.
2. Then add this response header:

```text
Content-Security-Policy: default-src 'self'; base-uri 'self'; object-src 'none'; frame-ancestors 'none'; form-action 'self' https://buttondown.com; img-src 'self' data:; font-src 'self'; style-src 'self'; script-src 'self'; upgrade-insecure-requests
```

If the Cloudflare HTML-injecting features must stay on, do not use that strict CSP. It will block the injected inline scripts/styles. A temporary compatibility CSP would need `script-src 'self' 'unsafe-inline'` and `style-src 'self' 'unsafe-inline'`, which is materially weaker.

## Verification

Commands used to verify the Cloudflare changes:

```bash
curl -I http://tendijournal.app/
curl -I http://twodesks.app/
curl -I https://tendijournal.app/
curl -I https://twodesks.app/
curl --tls-max 1.1 -I https://tendijournal.app/
curl --tls-max 1.1 -I https://twodesks.app/
```

Expected results:

- The HTTP URLs return a 301 or 308 redirect to HTTPS.
- HTTPS responses include the configured security headers.
- `Access-Control-Allow-Origin: *` is absent unless intentionally retained.
- TLS 1.1 and older clients fail with a protocol version error.
